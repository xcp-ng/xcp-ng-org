Easy To Install

Download 8.3 LTS Download 8.2 LTS Web UI Migrate from VMware Upgrade from XenServer Security and mirrors Old releases

XCP-ng 8.3 LTS

The current LTS release

Grab the latest 8.3 LTS full ISO (661MiB), then create your bootable USB key with:

dd if=xcp-ng-8.3.0-20250606.iso of=/dev/sdX bs=8M oflag=direct

Then just boot on it. Don't miss the release notes!

(⚠️ To upgrade from 8.2+XOSTOR, use the dedicated upgrade ISO)

On Windows, you can use Rufus to create the bootable USB stick. SHA256 full ISO: 

4d6f5a99da0d70920bc313470ad2b14decab66038f0863ca68a2b81126ee2977

We also offer a smaller Net install ISO (172MiB). SHA256 netinstall ISO: 

9a5dcc8d98949ee207d28307b8b94320d1ffd24841e34ca74e1c0f0422e5ecab

XCP-ng 8.2 LTS

The previous LTS release. Soon to reach EOL.

Grab the latest 8.2 LTS full ISO (590MiB), then create your bootable USB key with:

dd if=xcp-ng-8.2.1-20231130.iso of=/dev/sdX bs=8M oflag=direct

Then just boot on it. Don't forget to always read the release notes!

On Windows, you can use Rufus to create the bootable USB stick. SHA256 full ISO: 

108aa7144d1f5dd169a1b99ca70b510527136c549f2e3c3c1707b619e17ee1b6

You can also download the Net install ISO (160MiB), then create your bootable USB key in the same fashion. SHA256 netinstall ISO: 

6d5e221ac56d318436105e20513197cecfc8fa29eee4d9bf72b5fd52db345a02

Easy migration out of Broadcom VMware

Join the exodus with simple tools to assist on your VMware migration

Leverage Xen Orchestra to seamlessly migrate your VMware VMs to XCP-ng. This tool supports migration from any accessible ESXi host, including those at remote locations, by directly streaming the VMs to XCP-ng.

Learn more about the process by exploring our step-by-step guide on how to migrate from VMware to XCP-ng with just a few clicks.

Need further assistance?

Contact us for a personalized and dedicated migration plan tailored to your existing VMware infrastructure's transition to XCP-ng.

Upgrade from XenServer

Migrate out XenServer to enjoy a true open platform

Just insert the ISO and follow instructions. You'll have an upgrade panel, just select it and that's it!

After a reboot, you'll be on XCP-ng while keeping all your previous settings (Storage, VMs…)



No physical access to the XenServer host?

Please check our documentation for advanced upgrade cases.

Security

XCP-ng is focused on security, and it starts here

Security matters: all our repositories, ISOs and packages are signed with a GPG key. This key is available here:

https://xcp-ng.org/RPM-GPG-KEY-xcpng

Then, you can follow our guide on how to check authenticity of our ISO, and even any XCP-ng RPM repository or mirror.

Found a security problem in XCP-ng? Please email us on security at xcp-ng dot org

Manage XCP-ng at scale

Web UI, API or CLI, Xen Orchestra provides even native backup features.

Just go on this web page and follow instructions:

https://vates.tech/deploy

This will deploy a small Xen Orchestra VM doing all the management and backup for your XCP-ng. Follow instructions given on the screen, and you're up and running!

XCP-ng can be managed via both a centralized UI, API or a CLI. The choice is yours. There's also Terraform & Packer providers available.

If you want to learn more about managing XCP-ng, please read all the possibilities on our official documentation.
